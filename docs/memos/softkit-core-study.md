## SoftKit Core 学习备忘

- **多租户抽象设计**：`AbstractTenantResolutionService` 抽象了租户解析与用户所属校验两个关键职责，并通过泛型约束 JWT 载荷类型，提示我们在自研多租户方案时应提前定义统一接口，以便根据请求头、子域或 Token 等不同来源灵活扩展实现。
- **引导期容错工具**：`callOrUndefinedIfException` 将可能抛出异常的逻辑包装成“执行或返回异常对象”的形式，避免重复书写 try/catch，可在配置读取、环境探测、动态模块加载等易失败场景中复用同类模式。
- **启动流程模块化**：`setup-web-app.ts` 展示了将配置加载、插件注册、生命周期钩子拆分成小块再组装的引导流程，有助于我们在平台内核中构建可测试、可替换的启动管线。
- **配置集中与类型约束**：`app.config.ts` 将应用配置集中声明并结合启动流程注入，强调“集中声明 → 显式注入 → 强类型校验”的链路，值得在自有配置体系中借鉴。
- **适配项目规范注意事项**：SoftKit 的部分抽象缺少中文 TSDoc 注释，如直接采纳需补充符合本项目章程的中文注释，并结合 Clean Architecture + CQRS + ES + EDA 混合架构要求重新评估职责边界。
